# i18n.js

Translation system. Built around a tagged template literal `t`, a runtime `translate(...)` function, and an automatic DOM observer that translates any element carrying a `data-i18n` attribute.

**Import path from a third-party extension:**

```js
import { t, translate, applyLocale, addLocaleData, getCurrentLocale } from '../../../i18n.js';
```

## Exports at a glance

| Export | Kind | Summary |
|--------|------|---------|
| `t` | template tag | Translate a template literal in-place. |
| `translate` | function | Translate a plain string by text or key. |
| `applyLocale` | function | Apply translations to a DOM tree or HTML string. |
| `addLocaleData` | function | Inject extra translations for the active locale. |
| `getCurrentLocale` | function | Return the active locale ID. |
| `initLocales` | async function | Boot-time locale loader (called by ST). |

## Reference

### `` t`...` → string ``

Tagged template literal that translates interpolated strings. Values are replaced with `${0}`, `${1}`, … placeholders for the translation key, then re-substituted with the actual values after lookup.

```js
import { t } from '../../../i18n.js';

const name = 'Lyra';
toastr.warning(t`Tag ${name} not found.`);
// Translation key:  "Tag ${0} not found."
// French entry:     "Tag ${0} introuvable."
// Final output:     "Tag Lyra introuvable."
```

Always use `t` for user-visible strings in your extension — even if you only ship English; it costs nothing and lets others contribute translations later.

### `translate(text, key = null) → string`

Translate by explicit key, falling back to `text` when no translation is registered.

```js
import { translate } from '../../../i18n.js';

const label = translate('Save', 'my_ext.save_button');
// Looks up "my_ext.save_button" first, falls back to "Save".
```

If `key` is omitted, `text` itself is used as the lookup key. Use `t` for interpolated strings and `translate` for plain strings or when you need a stable namespaced key.

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `text` | `string` | — | The text to translate; also serves as the fallback when no translation is found. |
| `key` | `string \| null` | `null` | Optional explicit lookup key. When omitted, `text` itself is used as the key. |

**Returns:** `string` — the translated text, or `text` if no translation entry exists.

### `applyLocale(root = document) → Document | string`

Walk `root` and translate every element with a `data-i18n` attribute. Pass an HTML string to receive a translated HTML string back; pass a `Document` (the default) to translate in place.

```js
import { applyLocale } from '../../../i18n.js';

const html = `<div data-i18n="Hello">Hello</div>`;
const translated = applyLocale(html);
```

> A `MutationObserver` is installed at boot, so any `data-i18n` element you insert into the live document is translated automatically. You only need `applyLocale` for detached fragments or pre-render strings.

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `root` | `Document \| string` | `document` | DOM root to translate in place, or an HTML string to translate and return. |

**Returns:** `Document \| string` — the same shape as the input: a `Document` when given a `Document`, an HTML `string` when given a string.

### `addLocaleData(localeId, data) → void`

Merge additional `{ key: translation }` pairs into the active locale. Silently no-ops if `localeId` does not match the current locale, or if locale data has not yet loaded. Existing keys are not overwritten — built-in translations win.

```js
import { addLocaleData, getCurrentLocale } from '../../../i18n.js';

addLocaleData(getCurrentLocale(), {
    'my_ext.greeting': 'Hej!',
});
```

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `localeId` | `string` | — | Locale code such as `'fr-fr'` or `'zh-cn'`. Must match the current locale or the call is a no-op. |
| `data` | `Record<string, string>` | — | Additional `{ key: translation }` pairs to merge into the locale dictionary. |

**Returns:** none.

### `getCurrentLocale() → string`

Returns the active locale ID, e.g. `'en-us'` or `'fr-fr'`. Comes from `localStorage['language']` if set, else `navigator.language`, else `'en'`.

**Returns:** `string` — the active locale ID.

### `initLocales() → Promise<void>`

Loads the language list and the active locale file, installs the `data-i18n` MutationObserver, and registers a few debug helpers. Called by SillyTavern during boot; you do not need to call it.

**Returns:** `Promise<void>` — resolves once locale data is loaded and the observer is installed.

## Notes

- The mutation observer reacts to **added nodes** and **`data-i18n` attribute changes**. If you set `textContent` on an existing element, it will not be retranslated automatically — set the `data-i18n` attribute (or its value) instead, or call `applyLocale` on the subtree.
- Keys can target attributes using `data-i18n="[title]my_key"` (set the element's `title` from `my_key`). Multiple bindings can be separated with `;`.
